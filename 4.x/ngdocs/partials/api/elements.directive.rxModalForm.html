rxModalForm
directive in module elements

Description

Responsible for creating the HTML necessary for modal form

The <rx-modal-form> directive is helpful for providing a common format to forms inside modals (hence the name).

This directive also provides an 'autofocus' mechanism, which will move the keyboard focus cursor to the first 'tabbable' input available in the form.

Usage

as element:
<rx-modal-form
       title="{String}"
       [subtitle="{String}"]
       [is-loading="{Boolean}"]
       [submit-text="{String}"]
       [cancel-text="{String}"]
       [return-text="{String}"]
       [default-focus="{String}"]>
</rx-modal-form>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
titleString

Title of modal window

subtitle
(optional)
String

Subtitle of modal window

isLoading
(optional)
Boolean

True to show a spinner by default

submitText
(optional)
String

'Submit' button text to use.

(default: 'Submit')

cancelText
(optional)
String

'Cancel' button text to use.

(default: 'Cancel')

returnText
(optional)
String

'Return' button text to use.

(default: 'Return')

defaultFocus
(optional)
String

The default focus element. May be 'submit' or 'cancel'. If not provided, it'll default to the first 'tabbable' input available in the form.

Example

<rx-modal-form
    title="My Form"
    is-loading="true"
    submit-text="Yes!">
</rx-modal-form>
